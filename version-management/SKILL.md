# 本地版本管理

## 核心规则

只在会修改本地已有文件时使用本 skill。只创建全新文件、读取文件、分析文件、生成独立新稿且不覆盖已有文件时，不需要记录版本。

修改本地已有文件前，先确保当前状态可以回退。和用户沟通时使用普通人能理解的说法：

- 说“记录当前版本”“查看这次改了什么”“回到之前版本”。
- 除非用户主动询问技术细节，否则不要说 commit、branch、staging、HEAD、stash 等术语。
- 未经用户明确确认，不要丢弃、覆盖、重置或清理用户已有改动。
- 使用这个 skill 时，不要推送、同步、发布或修改远程仓库设置。
- 如果同时需要使用文档、PPT、图片、代码等其他 skill，本 skill 只做外层版本保护；其他 skill 继续负责具体编辑。

## 何时触发

使用本 skill：

- 用户让 AI 修改、润色、重写、整理、更新、替换、删除本地已有文件内容。
- 用户让 AI 修改本地已有 PPT、Word、Excel、PDF、代码、配置等文件。
- 用户说“恢复”“还原”“回退”“回到上一个版本”“撤回 AI 的修改”等。
- 用户问“这次改了什么”“AI 改了哪些文件”。

不要使用本 skill：

- 只创建全新文件，且不覆盖、不改动任何已有文件。
- 只阅读、总结、分析文件，不写入本地文件。
- 只生成一段可以复制的内容，不落盘修改文件。

## 没有 git 时

如果脚本提示没有找到本地版本记录工具，告诉用户：

> 你的电脑还没有安装用于版本管理的基础工具。要我帮你安装吗？安装后我就可以在改文件前后记录版本，方便之后回退。

只有用户同意后，才帮助用户安装 git。安装完成后，再继续记录版本。

## 记录当前版本

从将要修改的项目目录运行本 skill 自带脚本。脚本路径取决于当前 agent 如何安装 skill：使用实际安装位置里的 `version-management/scripts/checkpoint.sh`，不要假设固定目录。

如果正在本仓库中使用，可以运行：

```bash
bash version-management/scripts/checkpoint.sh
```

如果脚本提示当前文件夹还没有版本记录，先问用户：

> 这个文件夹还没有版本记录。我可以先记录当前状态，这样改错了可以回到现在这个版本。要我现在设置吗？

只有用户同意后，才能运行：

```bash
bash version-management/scripts/checkpoint.sh --init-approved
```

如果不是在本仓库中使用，把上面的脚本路径替换为当前安装位置中的脚本路径。

## 正常修改流程

1. 先判断任务是否会修改本地已有文件；如果只是新建文件，不使用本 skill。
2. 在真正修改本地文件前，运行记录当前版本的脚本，建议消息为“AI 修改前记录”。
3. 如果是首次设置版本记录，必须先询问用户。
4. 用户同意并记录当前版本后，再执行修改。
5. 修改完成后，或本轮对话中一个明确修改批次结束后，再运行一次记录当前版本的脚本，建议消息为“AI 修改后记录”。
6. 用简单语言总结改了哪些文件。
7. 告诉用户：如果结果不满意，可以要求“回到 AI 修改前的版本”；如果满意，也已经保留了“AI 修改后的版本”。

示例：

```bash
bash version-management/scripts/checkpoint.sh --message "AI 修改前记录"
# 执行文件修改
bash version-management/scripts/checkpoint.sh --message "AI 修改后记录"
```

如果不是在本仓库中使用，把示例里的脚本路径替换为实际安装路径。即使很多轮对话后继续修改同一个已有文件，也要在每一轮实际修改前后各记录一次。

## 查看这次改了什么

当用户问“这次改了什么”时，可以在内部使用这些命令检查改动：

```bash
git status --short
git diff --stat
git diff
```

对用户默认只总结：哪些文件新增了、哪些文件修改了、哪些文件删除了。不要直接贴大段差异，除非用户明确要求。

## 回到之前版本

恢复、还原、回退都应该触发本 skill。即使 agent 自己会使用底层工具恢复，也必须按这里的确认规则执行。回退可能会覆盖当前文件状态，所以恢复前必须谨慎：

1. 先确认用户想回到哪个版本。
2. 用简单语言说明会影响哪些文件。
3. 获得用户明确确认后再恢复。
4. 优先只恢复用户指定的文件或范围，不要扩大影响。

如果当前改动可能还有价值，先记录一个新的当前版本，再执行回退。底层命令只在 agent 内部使用；面向用户继续说“回到之前版本”。
