# 小白级版本管理 Skill

这是一个给本地 AI agent 使用的版本管理 skill，面向不熟悉 git、也不知道需要做本地文件版本管理的普通用户。

欢迎来交流～wx：Chitu_AI

以前自己改 PPT、文档或代码，一次通常只改一两页、几个字；即使改错了，也比较容易发现和撤回。Agent 不一样，它可能几秒钟重写整个文件，甚至同时修改很多文件，改动范围和速度都远远超过手工操作。

这个 skill 的作用很简单：**AI 修改你已有的本地文件前先记录一次，改完后再记录一次。**  
这样你既能回到 AI 动手前的版本，也能保留 AI 改完后的版本。

你不需要懂 git，也不需要记命令。你只要把下面这些话复制给本地 agent 即可。

## 适合什么情况

- 让 AI 修改已有 PPT、文档、代码、配置文件。
- 担心 AI 一次改太多，不知道怎么撤回。
- 想在 AI 动手前和改完后都留下清楚的版本记录。
- 想知道这次 AI 到底改了哪些文件。

如果只是让 AI 新建一个全新的文件，暂时不需要用这个工具。

## 安装这个工具

把本项目的 GitHub 链接发给本地 agent，让它帮你安装：

```text
请从这个 GitHub 仓库根目录安装本地版本管理 skill：https://github.com/chituai/version-management。skill 名称是 version-management；如果安装工具需要填写路径，请使用仓库根目录 `.`。安装后，以后修改本地已有文件前先记录一次，改完后再记录一次；如果只是新建文件，不需要记录版本。
```

## 如果电脑还没装 git

这个工具底层需要用到 git。你不需要自己研究怎么装，直接让本地 agent 处理：

```text
请先检查我的电脑有没有安装 git。如果没有，请问我要不要安装；我同意后再帮我安装，然后再启用本地版本管理功能。
```

## 第二步：写进当前项目的规则文件

安装后，还需要把版本管理规则写进当前项目的 agent 规则文件。这样多轮对话以后，AI 仍然会记得在修改已有文件前后记录版本。

把这句话发给正在当前项目里工作的本地 agent：

```text
请从 https://github.com/chituai/version-management 读取 templates/AGENTS.version-management.md，并把里面的版本管理规则合并到当前项目的 agent 规则文件里。根据你自己的规则文件习惯选择 AGENTS.md 或 CLAUDE.md；如果文件已经存在，不要覆盖原内容，只把版本管理规则合并进去。这个项目规则是必须配置的：它只需要在修改已有文件或恢复旧版本时触发，新建文件不用触发。
```

## 平时怎么用

你可以这样对本地 agent 说：

```text
修改这个已有文件前先记录当前版本，改完后也记录一下。
```

也可以说：

```text
帮我看看这次 AI 改了什么。
```

或者：

```text
帮我回到上一个版本。
```

## 你会看到什么

第一次在某个文件夹里修改已有文件时，agent 应该先问你：

```text
这个文件夹还没有版本记录。我可以先记录当前状态，这样改错了可以回到现在这个版本。要我现在设置吗？
```

你同意后，它才会开始设置并记录当前版本。

之后再让 AI 修改已有文件时，它会先记录当前版本，再开始修改；修改完成后，它还会再记录一次改完后的版本。

## 需要知道的边界

- 这是本地保护，不是云备份。
- 它不会自动上传你的文件。
- 它不会发布到 GitHub。
- 它不会在没问你的情况下丢弃你的已有改动。
